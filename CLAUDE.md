# AI Research WhatsApp Digest

**Repo:** github.com/vickey-kapoor/ai-research-whatsapp-digest

## Project Overview
Automated daily AI research digest that fetches academic papers (focusing on AI Agents & Reasoning), ranks by importance using AI, generates **simple ELI5-style summaries** (explain like talking to a kid or grandma), and sends the top research to WhatsApp.

## Architecture
```
main.py                  # Entry point - orchestrates the workflow
src/
├── research_fetcher.py  # Aggregates research from all sources (parallel)
├── news_ranker.py       # Ranks research using GPT-4o-mini
├── news_summarizer.py   # Generates ELI5 summaries
├── whatsapp_sender.py   # Sends messages via Twilio WhatsApp API
└── fetchers/            # Research source fetchers
    ├── __init__.py
    ├── arxiv_fetcher.py      # arXiv API (cs.AI, cs.LG, cs.CL categories)
    ├── huggingface_fetcher.py # Hugging Face Daily Papers
    ├── pwc_fetcher.py        # Papers With Code API
    └── blog_fetcher.py       # Google AI, DeepMind, Meta AI blogs (RSS)
.github/workflows/
└── daily-news.yml       # GitHub Actions - runs daily at 10:00 AM CST (16:00 UTC)
```

## Data Flow
```
fetch_ai_research() ─► rank_research() ─► summarize_research() ─► format_research_message() ─► send()
       │
       └─► parallel fetch from 4 sources ─► deduplicate by title (0.85 similarity) ─► sort by date
```

## Key Dependencies
- `requests` - API HTTP calls (Hugging Face, Papers With Code)
- `feedparser` - arXiv and RSS blog feed parsing
- `openai` - GPT-4o-mini for ranking and ELI5 summaries
- `twilio` - WhatsApp message delivery
- `python-dotenv` - Environment variable loading

## Environment Variables
All stored in `.env` (local) and GitHub Secrets (CI/CD):
- `TWILIO_ACCOUNT_SID` - Twilio console
- `TWILIO_AUTH_TOKEN` - Twilio console
- `TWILIO_WHATSAPP_NUMBER` - format: `whatsapp:+14155238886`
- `YOUR_WHATSAPP_NUMBER` - format: `whatsapp:+14083944615`
- `OPENAI_API_KEY` - required for AI ranking and ELI5 summaries

## ELI5 Summary Style Guide
Summaries should be understandable by a non-technical person (kid or grandma).

**Bad (too technical):**
> "This paper introduces a novel transformer architecture with sparse attention mechanisms achieving SOTA on GSM8K reasoning benchmarks."

**Good (ELI5):**
> "Scientists built a smarter AI that's much better at solving math word problems - like the ones kids get in school. It figured out a way to focus only on the important parts of a question instead of getting confused by extra details."

## Message Format
- Research-only digest (no news)
- Max 1600 characters for WhatsApp compatibility
- Descriptions truncated to 200 chars, summaries to 150 chars
- **ELI5 summaries**: Simple, jargon-free explanations (like explaining to a kid or grandma)
- Generated by GPT-4o-mini with simplified prompts

## Common Issues
1. **"Invalid From and To pair"** - WhatsApp numbers missing `whatsapp:` prefix or country code (`+1` for US)
2. **Messages not received** - User must join Twilio sandbox first by texting join code to Twilio number
3. **Missing modules** - Run `pip install -r requirements.txt`

## Security Measures
- **HTTPS only** - All API calls use HTTPS (arXiv, Hugging Face, PWC, blogs)
- **Prompt injection protection** - User input sanitized before AI prompts
- **URL validation** - URLs validated before including in messages
- **Error masking** - Sensitive details not exposed in error logs
- **Request timeouts** - 30s timeout on all network requests
- **Pinned dependencies** - Exact versions in requirements.txt
